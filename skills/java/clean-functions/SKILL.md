# Clean Functions

## F1: Too Many Arguments (Maximum 3)

```java
// Bad - too many parameters
User createUser(String name, String email, int age, String country,
                String timezone, String language, boolean newsletter) {
    ...
}

// Good - use a record
record UserData(
    String name,
    String email,
    int age,
    String country,
    String timezone,
    String language,
    boolean newsletter
) {}

User createUser(UserData data) {
    ...
}
```

More than 3 arguments means your method is doing too much or needs
a data structure.

## F2: No Output Arguments

Don't modify arguments as side effects. Return values instead.

```java
// Bad - modifies argument
void appendFooter(StringBuilder report) {
    report.append("\n---\nGenerated by System");
}

// Good - returns new value
String withFooter(String report) {
    return report + "\n---\nGenerated by System";
}
```

## F3: No Flag Arguments

Boolean flags mean your method does at least two things.

```java
// Bad - method does two different things
void render(boolean isTest) {
    if (isTest) {
        renderTestPage();
    } else {
        renderProductionPage();
    }
}

// Good - split into two methods
void renderTestPage() { ... }
void renderProductionPage() { ... }
```

## F4: Delete Dead Functions

If it's not called, delete it. No "just in case" code. Git preserves history.
